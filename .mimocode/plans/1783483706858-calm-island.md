# Plan: Preserve Registration Data Through Login Flow

## Goal
When a user submits the registration form without being logged in, instead of creating the account immediately, store the registration data in the session and show the login form. After the user logs in, automatically complete the registration.

## Current Flow
1. User fills registration form → submits POST → account created → auto-logged in → redirected to dashboard

## Desired Flow
1. User fills registration form → submits POST → validated → data stored in session → redirected to login
2. User logs in → pending registration data found in session → account created automatically → redirected to dashboard

## Files to Modify

### 1. `user/register.php`
**Change:** Instead of creating the account on POST, store validated data in `$_SESSION['pending_registration']` and redirect to login.

- Keep all existing validation (name, email, password, terms)
- On successful validation: store form data in `$_SESSION['pending_registration']` as an associative array
- Set a `$message` to inform the user that login is required
- Redirect to `login.php`
- Remove the auto-login and INSERT logic from the registration handler

### 2. `user/login.php`
**Change:** After successful login, check for `$_SESSION['pending_registration']` and auto-complete the registration.

- After successful authentication (after setting session vars), check if `$_SESSION['pending_registration']` exists
- If it exists:
  - Extract the stored registration data (name, email, password, role, phone)
  - Run the same username generation and INSERT logic (moved from register.php or called via a shared function)
  - Set `$_SESSION['user_id']` to the new user's ID
  - Clear `$_SESSION['pending_registration']`
- Show a notification banner on the login page if there is pending registration data (e.g., "Your registration data has been preserved. Please log in to complete registration.")

## Implementation Details

```php
];
```
