# Plan: Fix Donation History Flow

## Problem

Currently, `user/profile.php` directly inserts into `donation_history` when a user submits donor information from their profile page. This creates donation records **before** any blood request is made or approved, which is incorrect.

## Current Flow (Incorrect)

```
User Profile → Submit Donor Info → donor table + donation_history table (with request_id=0)
```

## Desired Flow (Correct)

```
User Profile → Submit Donor Info → donor table ONLY
Admin Dashboard → Assign Donor to Request → donation_history table (with actual request_id)
```
